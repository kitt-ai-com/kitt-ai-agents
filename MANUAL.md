## 1. 시스템 개요

### 이 시스템이 뭔가요?

회사의 모든 업무를 AI 에이전트가 팀별로 나누어 처리하는 시스템입니다.
각 팀(마케팅, 콘텐츠, 디자인, 개발, 이커머스 등)마다 전문 AI 에이전트가 있고,
CEO 에이전트가 이들을 총괄 관리합니다.

### 핵심 원리

Claude Code는 프로젝트 폴더의 `CLAUDE.md` 파일을 자동으로 읽습니다.
이 파일에 에이전트의 역할, 행동 규칙, 업무 프로세스, 참고 데이터를 적어두면
Claude가 해당 역할에 맞게 동작합니다.

**즉, CLAUDE.md = 에이전트의 두뇌**입니다.

### 조직도 구조

```
ai-company-agents/
│
├── CLAUDE.md                    ← CEO 오케스트레이터
├── README.md                    ← 프로젝트 설명
├── MANUAL.md                    ← 이 매뉴얼
│
├── strategic-hq/CLAUDE.md       ← 전략기획실 (21명 멀티에이전트)
├── marketing/CLAUDE.md          ← 마케팅팀
├── content/CLAUDE.md            ← 콘텐츠팀
├── design/CLAUDE.md             ← 디자인팀
├── development/CLAUDE.md        ← 개발팀
├── ecommerce/CLAUDE.md          ← 이커머스팀
├── finance-ops/CLAUDE.md        ← 재무/경영지원팀
└── agent-consulting/CLAUDE.md   ← 에이전트 컨설팅팀
```

## 2. 시작하기 (초기 세팅)

### 사전 준비

1. **Node.js 18 이상** 설치
   - https://nodejs.org 에서 다운로드

2. **Claude Code 설치**
   ```bash
   npm install -g @anthropic-ai/claude-code
   ```

3. **Anthropic API 키 설정** (첫 실행 시 자동 안내됨)

### 프로젝트 세팅

```bash
# 1. 다운받은 ai-company-agents 폴더를 원하는 위치에 놓기
# 예시: 홈 디렉토리에 놓기

# 2. 폴더로 이동
cd ai-company-agents

# 3. Claude Code 실행
claude

# 4. 잘 되는지 테스트
> "현재 조직도를 보여줘"
> "마케팅팀의 역할을 설명해줘"
```

### 첫 실행 시 확인할 것

- Claude가 "CEO 오케스트레이터" 역할로 응답하는지 확인
- 팀 배정이 제대로 되는지 테스트 질문 해보기
  - "광고 캠페인을 기획해줘" → 마케팅팀으로 배정되어야 함
  - "상세페이지 디자인 방향 잡아줘" → 디자인팀으로 배정되어야 함

## 3. 기본 사용법

### 방법 A: 루트에서 CEO에게 지시하기 (추천)

```bash
cd ai-company-agents
claude
```

CEO 오케스트레이터가 요청을 분석하고 적절한 팀에 배정합니다.

```
사용자: "새로운 건강식품 쇼핑몰을 런칭하려고 해. 전체 계획을 세워줘"

CEO 에이전트: 
→ 전략기획실: 시장 분석 및 전략 수립
→ 이커머스팀: 쇼핑몰 플랫폼 선정 및 세팅
→ 디자인팀: UI/UX 설계
→ 콘텐츠팀: 상품 상세페이지 기획
→ 마케팅팀: 런칭 광고 전략
→ 개발팀: 기술 구현
(순서대로 실행 계획을 제시)
```

### 방법 B: 특정 팀 폴더로 이동해서 직접 지시하기

```bash
cd ai-company-agents/marketing
claude
```

해당 팀 에이전트가 직접 응답합니다. 더 깊고 전문적인 답변을 받을 수 있습니다.

```
사용자: "메타 광고 캠페인을 기획해줘. 
        예산 월 300만원, 건강식품, 타겟 30~50대 여성"

마케팅 에이전트:
→ 캠페인 구조 설계
→ 타겟 오디언스 설정
→ 예산 배분 계획
→ 소재 방향 제안
→ 예상 성과 (ROAS, CPA 등)
```

### 방법 C: 전략기획실 풀 프로세스 실행 (중요 의사결정용)

```bash
cd ai-company-agents/strategic-hq
claude
```

```
사용자: "B2C SaaS 서비스를 런칭하려고 하는데, 
        진출해야 할지 검토해줘"

전략기획실:
→ Stage 1 (조사): 시장/경쟁/기술 리서치
→ Stage 2 (토론): 낙관론-비관론-현실론 등 다각도 토론
→ Stage 3 (종합): 최종 전략 보고서 + 실행 로드맵
```

## 4. 3가지 커맨드 체계: 질문 / 학습 / 기준

에이전트에 대한 모든 입력은 3가지로 구분됩니다. 이 구분이 매우 중요합니다.

### 왜 구분하나요?

| 유형 | 한 줄 요약 | 에이전트가 처리하는 방식 |
|---|---|---|
| **질문** | 업무를 시키는 것 | 기준을 지키며 + 학습을 참고해서 답변 |
| **학습** | 참고하면 좋은 노하우 | 상황에 따라 유연하게 활용 |
| **기준** | 절대 어기면 안 되는 규칙 | 모든 결과물에 강제 적용, 위반 시 자동 수정 |

**학습**은 "이렇게 하면 효과 좋더라"이고,
**기준**은 "이건 무조건 지켜야 해"입니다.

### 커맨드 목록

#### 업무 질문
| 커맨드 | 예시 |
|---|---|
| `/마케팅 [질문]` | `/마케팅 메타 광고 캠페인 기획해줘` |
| `/콘텐츠 [질문]` | `/콘텐츠 건강식품 상세페이지 카피 써줘` |
| `/디자인 [질문]` | `/디자인 인스타 광고 배너 방향 잡아줘` |
| `/개발 [질문]` | `/개발 랜딩페이지 만들어줘` |
| `/이커머스 [질문]` | `/이커머스 카페24 상품등록 도와줘` |
| `/재무 [질문]` | `/재무 이번 달 손익 분석해줘` |
| `/전략 [질문]` | `/전략 B2C 진출 검토해줘` |

#### 학습 등록 (노하우 추가)
| 커맨드 | 예시 |
|---|---|
| `/마케팅-학습 [내용]` | `/마케팅-학습 리뷰 카루셀이 단일 이미지보다 CTR 2배 높음` |
| `/콘텐츠-학습 [내용]` | `/콘텐츠-학습 네이버 블로그는 이미지 15장 이상이 SEO에 유리` |
| `/디자인-학습 [내용]` | `/디자인-학습 상세페이지 메인 이미지는 밝은 배경이 전환율 높음` |

#### 기준 등록 (필수 규칙 추가)
| 커맨드 | 예시 |
|---|---|
| `/마케팅-기준 [내용]` | `/마케팅-기준 건기식 광고는 식약처 심의 기준 필수 준수` |
| `/디자인-기준 [내용]` | `/디자인-기준 고객사 로고는 원본만 사용, 임의 변형 금지` |
| `/개발-기준 [내용]` | `/개발-기준 API 키는 절대 코드에 하드코딩 금지` |

#### 관리 커맨드
| 커맨드 | 기능 |
|---|---|
| `/마케팅-학습목록` | 등록된 노하우 전체 보기 |
| `/마케팅-기준목록` | 등록된 기준 전체 보기 |
| `/마케팅-학습삭제 [번호]` | 노하우 삭제 (누구나 가능) |
| `/마케팅-기준삭제 [번호]` | 기준 삭제 (**관리자만 가능**) |

### CLAUDE.md 안에서의 위치

각 팀의 CLAUDE.md에서 기준과 학습은 이렇게 구분되어 있습니다:

```markdown
## ⛔ 기준 (반드시 준수)
- 건기식 광고는 식약처 심의 기준 필수 준수
- '최고', '최초', '유일' 등 최상급 표현 사용 금지
  ↑ 모든 결과물에 강제 적용. 위반 시 자동 수정.

## 💡 학습 (참고 노하우)
- 리뷰 기반 카루셀이 단일 이미지보다 CTR 약 2배 높음
- 건강식품 전환율이 높은 시간대: 오후 9시~11시
  ↑ 참고 데이터. 상황에 따라 유연하게 활용.
```

## 5. 학습/기준 등록 검토 프로세스

학습이나 기준을 등록할 때 에이전트가 **무조건 바로 넣지 않습니다.**
반드시 검토 → 피드백 → 확인 과정을 거칩니다.

### 왜 검토가 필요한가요?

- 부정확한 정보가 들어가면 이후 모든 답변이 오염됨
- 기존 내용과 모순되는 내용이 들어갈 수 있음
- 너무 모호한 내용은 오히려 에이전트를 혼란시킴
- 특히 **기준**은 모든 결과물에 강제 적용되므로 잘못 넣으면 피해가 큼

### 검토 항목 (5가지)

| 항목 | 체크 내용 | 예시 |
|---|---|---|
| **유효성** | 사실에 부합하는가? | "메타 광고는 일예산 5만원 이상" → 업종마다 다름 |
| **구체성** | 실행 가능한 수준인가? | "좋은 소재를 만들어라" (X) → "첫 3초에 후킹 포함" (O) |
| **충돌 여부** | 기존 내용과 모순 없는가? | 기존: "A/B 테스트 필수" vs 새로: "소재 1개만 운영" |
| **범위** | 해당 팀에 맞는 내용인가? | 디자인 내용이 마케팅팀에 등록되면 안 됨 |
| **개선 가능성** | 더 나은 버전이 가능한가? | 에이전트가 더 정확한 표현으로 다듬어서 제안 |

### 실제 검토 흐름 예시

#### 학습 등록 예시

```
팀원: /마케팅-학습 메타 광고는 예산 하루 5만원 이상 써야 효과 있음

에이전트:
📋 학습 등록 검토 결과

✅ 유효성: 일부 맞음
   - 메타 광고 학습에 최소 예산이 필요한 건 사실입니다.
   - 단, "5만원"은 업종/타겟에 따라 크게 달라집니다.

📏 구체성: 부족
   - 어떤 업종, 어떤 캠페인 유형에서의 5만원인지 불명확합니다.

🔄 기존 내용과 충돌: 없음

📂 범위 적합성: 적합 (마케팅팀 업무 범위)

💡 개선 제안:
   - 원본: "메타 광고는 예산 하루 5만원 이상 써야 효과 있음"
   - 개선안: "메타 광고 최소 일예산은 타겟 CPA의 2~3배 이상 권장.
             건강식품 기준 일 5~10만원이 학습 효율적.
             패션은 일 3~5만원부터 데이터 수집 가능."
   - 이유: 업종별 기준을 분리하면 더 실용적으로 활용 가능

등록하시겠습니까?
1️⃣ 원본 그대로 등록
2️⃣ 개선안으로 등록 (추천)
3️⃣ 직접 수정 후 등록
4️⃣ 취소
```

#### 기준 등록 예시 (더 엄격)

```
팀원: /디자인-기준 모든 이미지는 PNG로 저장

에이전트:
📋 기준 등록 검토 결과

⚠️ 유효성: 문제 있음
   - 웹용 사진/배너는 JPG가 용량 면에서 유리합니다.
   - PNG는 투명 배경이 필요한 경우에만 권장됩니다.

📏 구체성: 양호하나, 적용 범위가 너무 넓음

🔄 기존 내용과 충돌: 없음

📂 범위 적합성: 적합 (디자인팀 업무 범위)

⚠️ 부작용 검토 (기준 전용):
   - 이 기준을 그대로 넣으면 모든 웹 배너/광고 소재도
     PNG로 만들게 되어 파일 크기가 불필요하게 커집니다.
   - 쇼핑몰 상세페이지 로딩 속도에 악영향을 줄 수 있습니다.

💡 개선 제안:
   - 원본: "모든 이미지는 PNG로 저장"
   - 개선안: "로고/아이콘/투명배경 → PNG.
             사진/배너/웹용 → JPG (품질 85% 이상).
             상세페이지 이미지 → JPG, 가로폭 860px 기준."
   - 이유: 용도별 포맷을 분리하면 품질과 성능을 모두 확보

등록하시겠습니까?
1️⃣ 원본 그대로 등록
2️⃣ 개선안으로 등록 (추천)
3️⃣ 직접 수정 후 등록
4️⃣ 취소
```

### 기준 vs 학습 검토 강도 차이

| 항목 | 💡 학습 검토 | ⛔ 기준 검토 |
|---|---|---|
| 유효성 | 확인 | 확인 |
| 구체성 | 확인 | 확인 |
| 충돌 여부 | 확인 | 확인 |
| 범위 | 확인 | 확인 |
| 개선 가능성 | 확인 | 확인 |
| **부작용 검토** | - | ✅ 추가 검토 |
| **관리자 승인** | 불필요 | 권장 |
| **삭제 권한** | 누구나 | 관리자만 |

## 6. 에이전트 다듬기 & 고도화

### Claude Code 안에서 대화로 수정하기

가장 쉬운 방법입니다. Claude Code에서 바로 지시하면 CLAUDE.md를 직접 수정해줍니다.

```bash
cd ai-company-agents
claude
```

## 7. 실제 업무 데이터로 학습시키기

"학습"이란 CLAUDE.md에 우리 회사만의 지식과 데이터를 쌓아가는 것입니다.
파인튜닝이 아니라, 에이전트가 참고할 정보를 계속 추가하는 방식입니다.

### 단계별 데이터 축적 가이드

#### 1단계: 기본 SOP 넣기 (첫 1주차)

각 팀이 실제로 하는 업무의 표준 프로세스를 넣습니다.

```
> "마케팅팀에 우리가 실제로 메타 광고 세팅할 때 쓰는 프로세스를 추가해줘:
   1. 비즈니스 관리자 접속
   2. 캠페인 만들기 > 목적: 전환
   3. 어드밴티지+ 캠페인 예산 ON
   4. 광고세트 > 전환위치: 웹사이트 > 픽셀 선택
   5. 타겟: 코어 오디언스 or 유사 오디언스
   6. 노출위치: 자동 노출위치 (초기) / 수동 (최적화 후)
   7. 소재 업로드 > 헤드라인, 설명문, CTA 버튼 설정
   8. 검토 후 게시"
```

```
> "이커머스팀에 카페24 상품등록 SOP를 추가해줘:
   1. 상품 촬영 완료 확인
   2. 카페24 관리자 > 상품 > 상품등록
   3. 상품명 작성 규칙: [브랜드명] 상품명 용량/수량
   4. 카테고리 선택 (대>중>소)
   5. 판매가/할인가/원가 입력
   6. 옵션 설정 (색상, 사이즈 등)
   7. 대표이미지 + 추가이미지 업로드
   8. 상세페이지 HTML 삽입
   9. SEO 설정 (메타 타이틀, 디스크립션)
   10. 배송 설정 확인 후 판매 시작"
```

#### 2단계: 실제 성과 데이터 넣기 (2~3주차)

실제 캠페인 결과, 성과 지표를 벤치마크로 넣습니다.

```
> "마케팅팀에 우리 실제 성과 벤치마크를 추가해줘:

   ## 내부 벤치마크 데이터
   
   ### 건강식품 카테고리
   - 메타 광고 평균 ROAS: 350~450%
   - 네이버 검색광고 평균 ROAS: 500~700%
   - 평균 CPA: 15,000~25,000원
   - 가장 성과 좋은 소재 유형: 리뷰 기반 카루셀
   - 전환율이 높은 시간대: 오후 9시~11시
   
   ### 패션 카테고리
   - 메타 광고 평균 ROAS: 250~350%
   - 인스타 릴스 평균 도달: 5,000~15,000
   - 가장 성과 좋은 소재 유형: 착장 영상"
```

#### 3단계: 실패/성공 경험 넣기 (지속적)

시행착오에서 얻은 노하우를 누적합니다.

```
> "콘텐츠팀에 우리가 경험한 주의사항들을 추가해줘:

   ## 실전 노하우 & 주의사항
   
   ### 상세페이지
   - gif 사용 시 카페24에서 로딩 심하게 느려짐 → 영상은 유튜브 임베드 권장
   - 모바일 기준 이미지 가로폭 860px이 최적
   - 텍스트 이미지보다 HTML 텍스트가 SEO에 유리
   
   ### 블로그
   - 네이버 블로그는 이미지 15장 이상이 체류시간과 SEO에 유리
   - 제목에 핵심 키워드를 앞쪽에 배치
   - 발행 후 24시간 내 수정하면 노출에 불리
   
   ### SNS
   - 인스타 릴스는 처음 3초에 후킹 없으면 이탈률 80%
   - 해시태그는 10~15개가 최적 (너무 많으면 스팸 판정)
   - 스토리는 하루 3~5개가 적당, 그 이상은 피로도 증가"
```

#### 4단계: 고객사별 정보 넣기 (고객 확보 시)

고객사마다의 특성과 요구사항을 에이전트가 기억하게 합니다.


## 11. 주간 운영 루틴

에이전트는 쓰면서 계속 개선해야 똑똑해집니다.
아래 루틴을 매주 따라하세요.

### 매주 금요일 30분 루틴

```
[체크리스트]

□ 1. 이번 주 반복 업무 체크
     → "이번 주에 같은 일을 2번 이상 했나?"
     → 있으면 해당 팀 CLAUDE.md에 SOP로 추가

□ 2. 새로 알게 된 노하우 기록
     → "이번 주에 새로 알게 된 팁이 있나?"
     → 있으면 해당 팀에 노하우/주의사항으로 추가

□ 3. 에이전트 오답 체크
     → "에이전트가 잘못 답변하거나 엉뚱한 답을 한 적 있나?"
     → 있으면 행동 규칙에 "하지 말 것" 추가
     → 또는 더 구체적인 지시사항 추가

□ 4. 고객사 정보 업데이트
     → "고객사 상황이 바뀐 게 있나?" (예산, 상품, 전략 변경)
     → 있으면 고객사 프로필 업데이트

□ 5. 참고 문서 정리
     → "이번 주에 만든 리포트나 문서 중 재활용할 것이 있나?"
     → 있으면 references 폴더에 저장
```

```
□ 에이전트 품질 테스트: 각 팀에 테스트 질문 던져보기
□ 새로 도입할 팀/역할 검토
```

## 12. 비개발자를 위한 확장 방안

현재 Claude Code는 터미널 기반이라 비개발자가 쓰기 어렵습니다.
팀원들도 쉽게 쓸 수 있게 하려면 아래 방법을 고려하세요.

### 방법 A: Claude.ai Projects 활용 (당장 가능)

1. claude.ai에서 Projects 기능 사용
2. 각 팀별 프로젝트 생성 (마케팅, 콘텐츠, 디자인 등)
3. 해당 CLAUDE.md 내용을 프로젝트 설명(Instructions)에 복사
4. 팀원을 프로젝트에 초대
5. 팀원은 웹 브라우저에서 바로 사용 가능

**장점**: 별도 개발 불필요, 즉시 사용 가능
**단점**: Claude Code만큼 파일 관리가 자유롭지 않음

### 방법 B: 슬랙 봇 연동 (구현 완료 ✅)

`slack-bot/` 폴더에 슬랙봇이 구현되어 있습니다.

**설치 및 실행:**
```bash
cd slack-bot
npm install
cp .env.example .env  # 토큰 설정
npm start
```

**사용법:**
```
@봇 마케팅 메타 광고 캠페인 기획해줘      → 팀 에이전트에 질문
@봇 mk 광고 기획해줘                     → 짧은 별칭 사용
@봇 마케팅-학습 리뷰 카루셀이 CTR 2배     → 학습 등록 (검토 플로우)
```

## 13. 트러블슈팅 & FAQ

### Q: Claude Code가 CLAUDE.md를 안 읽는 것 같아요

**확인할 것:**
- 현재 디렉토리에 CLAUDE.md가 있는지 확인 (`ls CLAUDE.md`)
- 파일명이 정확히 `CLAUDE.md`인지 확인 (대소문자 주의)
- Claude Code를 다시 실행해보기

### Q: 에이전트가 역할을 벗어나서 답변해요

**해결:**
- CLAUDE.md의 "행동 규칙"에 제한사항을 더 구체적으로 추가
- 예: "마케팅 외의 질문에는 '이 질문은 [해당팀]에 더 적합합니다'라고 안내해줘"

### Q: CLAUDE.md가 너무 길어졌어요

**해결:**
- 핵심 역할/규칙만 CLAUDE.md에 남기기
- 상세 가이드, 데이터, 템플릿은 references/ 폴더로 분리
- CLAUDE.md에는 "자세한 내용은 references/파일명.md 참고" 표시

### Q: 고객사 정보가 섞여서 답변돼요

**해결:**
- 고객사 프로필을 명확하게 구분 (### 고객사A, ### 고객사B)
- 질문 시 항상 고객사명을 명시하도록 행동 규칙에 추가
- "고객사가 명시되지 않은 요청에는 어떤 고객사인지 먼저 확인할 것"

### Q: 여러 팀이 협업해야 하는 업무는 어떻게?

**해결:**
- 루트(CEO 오케스트레이터)에서 지시하면 자동으로 팀 간 배정
- 또는 직접 순서를 지정: "마케팅팀이 전략을 먼저 잡고, 콘텐츠팀이 소재를 만들어줘"

### Q: 에이전트에게 파일을 보여주고 싶어요

**해결:**
Claude Code에서 파일을 바로 읽을 수 있습니다:
```
> "marketing/references/리포트.md 파일을 읽고 개선점을 알려줘"
> "이 엑셀 파일을 분석해줘: /path/to/data.xlsx"
```
